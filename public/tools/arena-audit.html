What Arena Audit does

    

      Most AI agents fail audits for a boring reason: the brief was thin. Arena Audit turns your
      target — a URL, an app, a repo, even a dropped folder of source files — into a complete,
      structured audit brief. Fifteen evaluation categories. Fourteen expert roles. One Markdown
      prompt your agent can execute immediately.
    


    
How it works

    
    
      
  1. Point it at anything. Paste a website, store listing, or package name — or drop up to 300 local files. Type detection and fail-safe defaults fill every blank.
    2. 
      
  2. Shape the audit. Toggle categories (SEO, WCAG 2.2, Core Web Vitals, security posture…), pick a depth from focused to forensic, and watch the brief rebuild live.
    3. 
      
  3. Hand it off. One click copies the prompt. Paste it into Arena Agent Mode and the multi-disciplinary audit starts with full context — no re-explaining.
    4. 
    


    

      Private by design. Everything runs in your browser. Files you drop are read
      locally and never uploaded — only a compact evidence manifest (file names, sizes, detected
      type) enters the prompt text itself. No cookies, no analytics, no accounts.
    


    
Frequently asked questions

    

      Does Arena Audit run the audit itself?
      
No — it builds the brief. The audit runs in your AI agent (e.g. Arena Agent Mode), which keeps your data local and your agent in control.

    

    

      Is it free?
      
Yes. MIT-licensed, no account, no tracking.

    

    

      What can it audit?
      
Websites and web apps, iOS/Android apps, desktop software, SaaS products, APIs, browser extensions, and local codebases via file drop.

    

    

      Why 15 categories?
      
Each maps to a professional discipline (UX, brand, technical SEO, security…). Skipping categories produces faster, narrower briefs.

    

    

      Where do my files go?
      
Nowhere. Processing is 100% client-side; only a manifest of names and sizes is embedded in the generated prompt.
