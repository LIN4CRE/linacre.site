# Website Prompts — Linacre.site

Use these prompts for Cursor, v0, Lovable, Bolt, Relume, Framer, Figma AI, or any code/design assistant. Paste the **Master Brand Prompt** first.

## 1. Full Website Refresh Prompt

### What this is for
Use this when you want an AI coding tool to review and polish the whole site while keeping the current identity.

### Prompt

> Review and improve the Linacre.site website as a premium full-stack and AI engineer portfolio. Keep the existing core identity: amber, hexagon, pulse, cyber, terminal details, toolkit directory, roadmap, AI Lab, changelog, and private dashboard feeling. Make the design calmer, softer, more retro-modern, and more premium. Improve spacing, type hierarchy, card design, responsive layout, accessibility, hover states, and visual consistency. Add subtle amber glow and hex grid details without clutter. Use deep ink backgrounds, warm amber accents, cream text, and rare cyan/violet AI accents. Keep performance high and avoid unnecessary dependencies. The result should feel like a polished React/TypeScript engineering workspace with a memorable personal signature.

### Output requested

Ask the tool to provide:

- design audit,
- updated component plan,
- exact code changes,
- responsive notes,
- accessibility fixes,
- final QA checklist.

## 2. Homepage Hero Prompt

### What this is for
The hero section is the first brand impression. It must explain who you are and what the site does quickly.

### Prompt

> Design a homepage hero for Linacre.site. The hero should introduce David Linacre as a Full-Stack & AI Engineer and describe the site as a toolkit, roadmap, AI lab, and build archive. Visual style: deep ink background, soft amber radial glow, subtle hexagon grid, terminal command line microcopy, polished cards, calm retro-modern cyber. Include a strong headline, a one-sentence value proposition, two primary CTAs, and a small signature identity line: “amber · hexagon · pulse · cyber”. Add a minimal animated pulse element that feels premium and not distracting.

### Suggested copy

Headline options:

- Full-stack and AI engineering, built with calm systems.
- Practical tools, AI experiments, and full-stack craft.
- A quieter engineering hub for modern builders.

CTA options:

- Explore the toolkit
- Try the AI Lab
- View the roadmap
- Follow the changelog

## 3. Toolkit Directory Prompt

### What this is for
The toolkit is a key reason people will return to the site.

### Prompt

> Improve the toolkit directory UI for Linacre.site. It should feel like a curated engineering console, not a generic link list. Each tool card should show name, category, free/trial status, short value statement, link, and optional “my stack” badge. Add clear search, category chips, keyboard-friendly filtering, hover states, and empty states. Use amber borders and glow sparingly. Make the cards scannable, responsive, and accessible. Add small terminal-style hints such as “/ to search” and “showing 42 of 42 tools”.

### Quality goals

- Easy to scan in 5 seconds.
- Useful on mobile.
- No excessive animation.
- Clear focus states for keyboard users.

## 4. AI Lab Prompt

### What this is for
The AI Lab should feel experimental but safe, clear, and credible.

### Prompt

> Design or refine an AI Lab section for Linacre.site. It should feel like a private engineering sandbox with a clean terminal/chat interface. Include provider status, model selector, secure-key note, prompt input, conversation area, and clear disclaimers. Visual style: deep slate panels, amber focus states, rare cyan signal lights, mono labels, subtle pulse animation. Make it trustworthy and not gimmicky. The user should understand that API keys are handled securely server-side where applicable.

### Include

- empty state message,
- loading state,
- error state,
- privacy/security note,
- keyboard shortcuts,
- example prompts.

## 7. CSS Token Integration Prompt

### What this is for
Use when asking an AI coding tool to apply the brand colours and tokens.

### Prompt

> Integrate the Linacre.site brand tokens into the existing CSS/Tailwind system. Use ink backgrounds, deep slate cards, amber accents, warm cream text, subtle border colours, focus rings, hover glows, and mono metadata styling. Create reusable CSS variables for brand colours, radii, shadows, motion durations, and gradients. Ensure AA contrast for body text and visible keyboard focus states. Do not make the site visually noisy.

## 8. Website Negative Prompt

> Avoid huge animated backgrounds, unreadable low contrast, generic SaaS sections, stock photos, too many badges, overdone glassmorphism, harsh neon blue, constant glitch effects, excessive shadows, tiny text, slow scripts, and anything that weakens performance or accessibility.
