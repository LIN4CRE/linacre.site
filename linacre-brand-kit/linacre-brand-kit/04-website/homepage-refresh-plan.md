# Homepage Refresh Plan — Linacre.site

## Goal

Make the homepage feel like a premium personal engineering hub, not just a landing page.

It should answer:

1. Who is David Linacre?
2. What is Linacre.site useful for?
3. What can visitors do next?
4. Why does the brand feel memorable?

## Recommended Homepage Structure

### 1. Hero

Purpose: immediate clarity.

Include:

- name,
- role: Full-Stack & AI Engineer,
- one-sentence value proposition,
- CTAs,
- signature identity line,
- subtle pulse/hex visual.

Example:

> Full-stack and AI engineering, built with calm systems.

Subtext:

> Linacre.site is my toolkit directory, AI lab, roadmap, and changelog-driven workspace for modern web builders.

### 2. Toolkit Preview

Purpose: make the site useful immediately.

Include:

- search/filter UI,
- 6–8 featured tools,
- “view all tools” CTA,
- “my stack” badge.

### 3. AI Lab Preview

Purpose: show experimentation and technical credibility.

Include:

- chat/terminal visual,
- provider labels,
- privacy note,
- example prompts,
- link to open the lab.

### 4. Roadmap Preview

Purpose: serve self-taught developers.

Include:

- steps/stages,
- practical outcomes,
- recommended tools,
- beginner-friendly language.

### 5. Featured Builds / Projects

Purpose: proof of work.

Include:

- Linacre.site current version,
- GitHub links,
- technical stack,
- screenshots or cards,
- short explanation of what changed.

### 6. Changelog

Purpose: show momentum.

Include:

- latest release,
- version tags,
- concise technical highlights,
- link to full changelog.

### 7. Footer

Purpose: final navigation and credibility.

Include:

- contact,
- GitHub,
- LinkedIn,
- email,
- sitemap,
- version,
- small brand sentence.

## Homepage Design Notes

- Use dark backgrounds and amber accents.
- Give every section breathing room.
- Use mono metadata labels sparingly.
- Make cards visually consistent.
- Keep mobile layout simple.
- Do not hide core content behind too much animation.

## Performance Rules

- Optimise images.
- Prefer SVG for decorative graphics.
- Avoid massive animation libraries unless needed.
- Lazy-load non-critical sections.
- Test Lighthouse, accessibility, and mobile.

## Accessibility Rules

- Use semantic headings.
- Ensure visible focus styles.
- Keep contrast readable.
- Respect reduced motion.
- Make filters keyboard usable.
- Do not rely on colour alone for status.
